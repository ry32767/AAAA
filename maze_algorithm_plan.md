# 難しい迷路を作成するためのアルゴリズム検討

難しい迷路を作る場合、まず「**何をもって難しいとするか**」を定義するのが重要です。単にランダムに複雑にするだけだと、見た目は複雑でも解くのは簡単、という迷路になりがちです。

## 1. 難しさの主要指標

迷路の難易度は、主に次の要素で決まります。

| 指標 | 説明 |
|---|---|
| 最短経路長 | スタートからゴールまでの正解ルートが長い |
| 分岐数 | 解く途中で選択肢が多い |
| 行き止まり数 | 誤った道に入りやすい |
| 行き止まりの深さ | 間違えたときに戻る距離が長い |
| 正解ルートの見通しにくさ | 直線的でなく、頻繁に曲がる |
| ループの有無 | 戻ってきたことに気づきにくい構造になる |
| 局所的な対称性 | 似た形が多く、現在位置を把握しにくい |

「紙の迷路」「ゲーム内迷路」「自動生成ダンジョン」では最適な設計が少し変わりますが、まずはこの指標で評価できます。

---

## 2. 基本アルゴリズム候補

### A. 深さ優先探索法 DFS / Recursive Backtracker

迷路生成で最も有名な方法です。

特徴：

- 実装が簡単
- 長い通路ができやすい
- ループのない完全迷路になる
- 見た目が自然
- ただし分岐が少なく、慣れると解きやすい

完全迷路とは、任意の2点間の経路が必ず1本だけ存在する迷路です。ループがないため、壁伝い法でも必ず解けます。

難しい迷路のベースとして使えますが、そのままだと「長いだけ」の迷路になりやすいです。

---

### B. Prim法

未訪問セルをランダムに接続していく方式です。

特徴：

- 分岐が多くなりやすい
- 通路が細かく入り組む
- DFSより探索感が強い
- 正解ルートが短くなることもある

「選択肢の多さ」で難しくしたい場合に向いています。

---

### C. Kruskal法

全セルを独立集合として扱い、壁をランダムに壊して連結していく方法です。

特徴：

- 均質で自然な迷路になる
- 完全迷路を作りやすい
- Union-Find が必要
- 難易度制御はやや工夫が必要

構造の偏りが少ない迷路を作りたい場合に向いています。

---

### D. Wilson法 / Aldous-Broder法

ランダムウォークを使って、一様な全域木として迷路を作る方法です。

特徴：

- 統計的に偏りの少ない迷路を生成できる
- 自然で予測しづらい構造になる
- 実装や処理時間はやや重い
- 難易度調整は別途必要

理論的にきれいな迷路を作るなら有力ですが、「難しい迷路を狙って作る」には追加評価が必要です。

---

## 3. 難しい迷路を作るなら推奨方針

単に1回生成して終わりではなく、次のような流れがよいです。

```text
1. 迷路をランダム生成する
2. スタートとゴール候補を決める
3. BFSで最短経路を計算する
4. 難易度スコアを計算する
5. スコアが低ければ再生成する
6. 条件を満たす迷路だけ採用する
```

つまり、生成アルゴリズムだけで難しさを保証するのではなく、**生成後に評価して選別する**方式が堅実です。

---

## 4. 難易度スコアの設計

例えば、次のようなスコアを使えます。

```text
difficulty =
  1.0 * shortest_path_length
+ 2.0 * wrong_branch_count
+ 1.5 * average_deadend_depth
+ 1.0 * turn_count_on_solution
- 3.0 * straight_visibility_score
```

各要素の意味は以下です。

| 項目 | 意味 |
|---|---|
| shortest_path_length | 正解ルートの長さ |
| wrong_branch_count | 正解ルート上に接続された誤分岐の数 |
| average_deadend_depth | 行き止まりまでの平均距離 |
| turn_count_on_solution | 正解ルート上の曲がり角数 |
| straight_visibility_score | ゴール方向に一直線で見通せる度合い |

特に重要なのは、**正解ルート上の誤分岐**です。

スタートからゴールまでの道の途中に、深い行き止まりが多数あると、プレイヤーは頻繁に判断を迫られます。

---

## 5. 迷路タイプ別のおすすめ

### ループなし迷路を作りたい場合

おすすめ：

```text
DFS生成 + BFS評価 + 高スコア迷路だけ採用
```

理由：

- 実装が簡単
- 破綻しにくい
- スタート・ゴール間の最短経路を制御しやすい
- 紙の迷路に向く

### ループあり迷路を作りたい場合

おすすめ：

```text
完全迷路生成 + 一部の壁を追加で壊す + 難易度評価
```

ただし注意点があります。

ループを増やすと見た目は複雑になりますが、必ずしも難しくなるとは限りません。むしろ、ゴールへのショートカットができて簡単になる場合があります。

そのため、壁を壊すたびに最短経路を再計算し、難易度が下がりすぎないか確認する必要があります。

---

## 6. 実装上の推奨設計

まずは以下の構成が扱いやすいです。

```text
MazeGenerator
  - generate_by_dfs()
  - generate_by_prim()

MazeAnalyzer
  - shortest_path_bfs()
  - count_deadends()
  - count_branches_on_solution()
  - count_turns()
  - compute_difficulty_score()

MazeOptimizer
  - generate_many()
  - select_best()
```

最初から高度なアルゴリズムを作るより、**たくさん生成して、評価関数で最も難しいものを選ぶ**方が安定します。

---

## 7. 推奨アルゴリズム案

最初の実装としては、これが最もバランスが良いです。

```text
1. DFSで完全迷路を生成
2. 各セルから最も遠いセルをBFSで探す
3. そのセルからさらに最も遠いセルを探す
4. その2点をスタート・ゴールにする
5. 正解ルートをBFSで取得
6. 正解ルート上の分岐数・行き止まり深さ・曲がり角数を評価
7. 何百〜何千回か生成して、最高スコアの迷路を採用
```

この方法の利点は、スタートとゴールが迷路内でかなり遠くなりやすいことです。

グラフ理論的には、迷路を「セルを頂点、通路を辺とするグラフ」と見なし、BFSで木の直径に近い2点を探す方法です。完全迷路は木構造なので、この発想と相性が良いです。

---

## 8. 結論

難しい迷路を作るなら、生成アルゴリズム単体ではなく、

```text
ランダム生成
→ BFSで解を解析
→ 難易度スコアを計算
→ 高スコア個体を採用
```

という方式が適しています。

最初の候補としては、

```text
DFS生成 + BFS解析 + スコア選別
```

を推奨します。

次の段階では、セルサイズ、迷路サイズ、ループの有無、出力形式、難易度指標を決めると、実装設計に進めます。
