# 迷路ゲーム / 迷路ソルバー仕様書

## 1. 概要
本プロジェクトは「多階層の迷路を自動生成し、複数の解法アルゴリズムが解く様子を可視化する」アプリケーションである。

現在は次の 2 つの実装を持つ。

- **ブラウザ版（メイン）**: `index.html` / `styles.css` / `app.js`。GitHub Pages で公開する静的 Web アプリ。
- **Python 版（レガシー）**: `maze_game.py`。Tkinter による初期実装。参考用に保持する。

公開 URL: https://ry32767.github.io/MazeCreator/

---

## 2. ブラウザ版の機能

### 2.1. 迷路の自動生成
- 起動時および「生成」ボタンで、指定サイズ・指定階層数の迷路を生成する。
- 生成アルゴリズムは穴掘り法（深さ優先探索）。各階層は必ず全通路が連結した完全迷路になる。
- 階層が複数ある場合は、上下階の同一座標が通路同士の地点に「階段」を配置し、階をまたぐ移動を可能にする。
- スタートとゴールは BFS による最遠点探索で決め、スタートは最下階・ゴールは最上階に置く。ゴールは行き止まりになるよう調整する。
- 必ずスタートからゴールまで到達可能な経路を 1 つ以上持つ。

### 2.2. 表示
- Canvas に迷路を描画する。壁・通路・探索済み・経路などを色分けする。
- スタート（S）・ゴール（G）・階段（上り▲／下り▼／両方↕）は、セルが小さくても視認できるよう大きな円形マーカーで強調する。導き出した正解ルートは、暗い縁取り付きの太い線で連続的に描く。
- 表示モードを「全階（平面・縦並び）」「全階（3D俯瞰）」「1階ずつ」から選べる。複数階では既定で全階を同時表示するため、アニメーション中に階を行き来しても表示が切り替わらず追いやすい。
    - **全階（平面）**: 各階を縦に並べて真上から表示。
    - **全階（3D俯瞰）**: 各階を斜めから見たオブリーク投影で積層表示し、階段は上下階をつなぐ縦の柱で立体的に描く。
    - **1階ずつ**: 従来表示。「下階／上階」ボタンまたは PageUp / PageDown で階を切り替える。

### 2.3. 解法アルゴリズム（可視化）
次の 6 種類を選択して実行でき、探索の進行をアニメーション表示する。
- Paintbrush（塗りつぶし＋連結性判定による経路確定）
- 右手法 / 左手法（壁伝い）
- BFS（幅優先探索、最短経路を保証）
- DFS（深さ優先探索）
- A*（マンハッタン距離 + 階層差を許容的ヒューリスティックとし、最短経路を保証）

実行中の探索セル数・経路長・所要時間をメトリクスとして表示する。

### 2.4. 手動プレイ
- 矢印キー / WASD、または画面の方向パッドでプレイヤーを操作する。
- 壁は通り抜けられない。階段地点では自動で階を移動する。
- ゴール到達時に「ゴール」と表示する。

### 2.5. 研究・実験向け機能
- **シード指定**: 擬似乱数（mulberry32）にシードを与えることで、同じ設定なら同一の迷路を再現できる。空欄ならランダムなシードを割り当て、使用したシードを表示する。同一迷路上で複数アルゴリズムを公平に比較するための土台。
- **ループ率（braiding）**: 完全迷路の行き止まりを一定割合で開放し、別解（ループ）を作る。0% は唯一解、値を上げるほど壁伝い法が解けなくなり BFS/A* との差が際立つ。
- **階段密度 / 生成試行回数**: 階のつなぎ目あたりの階段数、最長経路を選ぶための生成試行回数を調整できる。
- **全解法比較**: 現在の迷路上で全解法を即時実行し、経路長・探索セル数・所要時間・最短一致を一覧表で比較する。
- **バッチ検証**: 現在の設定でランダムな迷路を複数生成し、各解法の平均経路長・平均探索セル数・最短率・平均所要時間を集計する。単一の迷路に依存しない統計的な比較ができる。処理は時間予算ごとに分割（非同期チャンク化）して実行し、進捗バーを表示するため、大きい迷路・多数の試行でも画面が固まらない。停止ボタンで中断できる。
- **探索ヒートマップ**: 解法の探索済みセルを「探索された順番」のグラデーション（寒色→暖色）で塗り分け、BFS の一様拡散と A*/DFS の偏った探索の違いを視覚化する。
- **パラメータ掃引**: ループ率または階段密度を横軸に振りながら、各解法の平均探索セル数を折れ線グラフ（縦軸）で比較する。パラメータが探索効率へ与える影響を一目で把握できる。
- **結果エクスポート**: バッチ検証・パラメータ掃引の結果を CSV / JSON で書き出し、外部ツール（表計算・解析スクリプト等）で再分析できる。JSON には生成設定のメタ情報を含む。

### 2.6. アクセシビリティ
- 操作状況は `aria-live` のステータス領域に反映する。
- 解法完了・生成・比較の結果は、スクリーンリーダー向けの詳細アナウンス領域（経路長・最短との差・探索セル数・所要時間など）に文章で出力する。
- 生成オプションには補足説明（`aria-describedby`）を付与している。

## 3. 技術仕様

### 3.1. ブラウザ版
- 使用言語: JavaScript（フレームワーク・ビルド不要）
- 描画: HTML5 Canvas
- データ構造:
    - 迷路: 階層ごとの 2 次元配列（0=通路, 1=壁）の配列 `layers[z][y][x]`
    - 階段: 座標キー `"x,y,z"` から接続先配列への `Map`
    - 座標: `{ x, y, z }` オブジェクト
- テスト: Node.js 標準テストランナー（`npm test`）。`app.js` は Node からも読み込め、DOM 非依存のアルゴリズム関数を検証できる。

### 3.2. Python 版（レガシー）
- 使用言語: Python 3 / `tkinter`
- 迷路生成: 穴掘り法（深さ優先探索）
- 操作: 矢印キー
